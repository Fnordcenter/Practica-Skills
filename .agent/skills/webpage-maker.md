# Webpage Maker Skill

## When to use this skill
- Whenever a user asks to "build a webpage," "make a website," or "create a frontend application."
- When creating a user interface that specifically requires standard HTML, CSS, and JavaScript.
- When generating landing pages, dashboards, or any UI where raw web technologies are preferred.

## How to use it
1. **Understand Requirements & Design:**
    - Clarify the user's requirements and the intended look and feel.
    - Plan the layout and semantic structure.
    - Prioritize a visually stunning, responsive, and modern design (e.g., using curated color palettes, dark modes, gradients, shadows, and modern typography).

2. **Scaffold the HTML (`index.html`):**
    - Set up a standard HTML5 boilerplate with correct meta tags for viewport and SEO.
    - Organize the content semantically (`header`, `main`, `section`, `footer`).
    - Link to an external CSS file (`<link rel="stylesheet">`) and a deferred JavaScript file (`<script defer>`).

3. **Implement Styling (`styles.css` or `index.css`):**
    - Use vanilla CSS to implement the core design system. Do not use TailwindCSS unless explicitly requested.
    - Use CSS variables for colors, spacing, and typography to maintain consistency.
    - Implement responsive layouts using CSS Flexbox and Grid.
    - Include smooth hover effects, micro-animations, and transitions to make the UI feel dynamic.

4. **Add Interactivity (`script.js` or `main.js`):**
    - Select DOM elements appropriately and add necessary event listeners.
    - Implement interactive features such as mobile menus, form handling, or dynamic content rendering.
    - Keep scripts focused, modular, and maintainable.

5. **Review and Polish:**
    - Verify that the layout looks impressive and does not break on different screen sizes.
    - Ensure good accessibility (ARIA attributes, keyboard navigation, color contrast).
    - Provide clear instructions to the user on how to run or view the created webpage (e.g., opening the HTML file in a browser).
